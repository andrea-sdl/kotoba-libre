# Toro Libre (Tauri, macOS-first)

Native desktop wrapper for `https://chat.example.com` (private Toro Libre), with:

- Native app window embedding `chat.example.com`
- Custom deep links (`torolibre://...`) with single-instance routing
- Manual preset management (agents/links) in Settings
- Spotlight-style launcher opened by global shortcut
- Unsigned internal macOS packaging (`.app`, `.dmg`, `.zip`)

## Core Features

- Main window loads remote `https://chat.example.com`
- Host-restriction policy defaults to `chat.example.com` only
- Settings window stores:
  - Global shortcut (default: `Alt+Shift+Space` on macOS, `CommandOrControl+Shift+Space` otherwise)
  - Open-in-new-window behavior
  - Host restriction toggle
  - Default launcher preset
- Presets stored locally:
  - `name`, `urlTemplate`, `kind`, `tags`
  - Dynamic query injection (`query` parameter appended automatically when provided)
- Launcher:
  - Fuzzy search over preset name/tags
  - Optional query text
  - Keyboard-first (up/down/enter/esc)

## Deep Link Contract

Supported now:

- `torolibre://open?url=<encoded_url>`
- `torolibre://preset/<presetId>?query=<encoded>`
- `torolibre://settings`

Universal-link-ready parsing is also implemented for:

- `https://chat.example.com/app/open?...`
- `https://chat.example.com/app/preset/<id>?...`
- `https://chat.example.com/app/settings`

Note: true macOS universal links still require Apple associated-domain entitlements and `chat.example.com` AASA hosting.

## Automated Releases (GitHub Actions)

This repo includes a macOS unsigned release workflow at `.github/workflows/release.yml`.

### Trigger modes

- Tag push: push a semantic version tag like `v0.2.0`
- Manual: run the `Release macOS (Unsigned)` workflow with:
  - `version` (for example `0.2.0` or `v0.2.0`)
  - optional `prerelease` and `draft` flags

### What it does

1. Validates version consistency across:
   - `package.json`
   - `src-tauri/Cargo.toml`
   - `src-tauri/tauri.conf.json`
2. Builds unsigned macOS artifacts
3. Generates `dist-artifacts/SHA256SUMS.txt`
4. Publishes a GitHub Release with:
   - `Toro Libre-unsigned.dmg`
   - `Toro Libre-unsigned-app.zip`
   - `SHA256SUMS.txt`

### Versioning contract

Before creating a release tag, keep all app version files aligned:

- `package.json`
- `src-tauri/Cargo.toml`
- `src-tauri/tauri.conf.json`

You can validate locally with:

```bash
./scripts/ci/validate-version.sh v0.2.0
```

If a release already exists for a tag, the workflow fails early with an actionable message.

## Notes for Distribution

Because the build is unsigned/not notarized:

- macOS Gatekeeper warnings are expected.
- Internal users may need to open via Finder context menu (`Open`) the first time.

## Icons

- Bundled desktop icons come from `src-tauri/icons`, generated by:

  ```bash
  cd src-tauri
  npx --yes tauri icon ../src/assets/icons/feather-icon-1-geometric.svg
  ```

- The runtime settings switch updates the macOS dock icon from the selected entry (`geometric`, `organic`, `bold`) using the platform AppKit API.

## Data Storage

Settings and presets are persisted via `tauri-plugin-store` in app data files:

- `settings.json`
- `presets.json`
